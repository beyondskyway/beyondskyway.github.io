SAE中MySQL备份与恢复

  • Cron
  • MySQL
  • Python
  • SAE
编程

手动备份：

由于SAE的备份和恢复执行都有一定的缓冲时间，对于突发情况不能等待，就需要手动去执行，DeferredJob的管理界面可以直接备份整个数据库的数据，并且提供下载。恢复的时候可以直接使用phpMyAdmin来恢复（数据量小于8,192 KB）

（1）解压DeferredJob上下载下来的数据包
（2）添加扩展名sql
（3）删除所有表的LOCK和UNLOCK语句（SAE用来避免数据库数据写入冲突语句，也就意味着我们在导入的时候不能对数据库进行写操作）
（4）phpMyAdmin导入sql文件
当然也可以直接使用phpMyAdmin导出数据然后在导入。

自动备份

自动备份需要用到DeferredJob（数据备份）、Cron（定时任务）和Mail（邮件发送）

配置信息

需要修改username，password等对应位置

修改model建任务记录表back_up

记录任务ID，表名,状态和时间（数据库方面使用的是SQLAlchemy+Mysql）

邮件发送（详情参考SAE的Mail）

数据备份

添加DeferredJob任务，根据返回信息添加备份信息到back_up表

值得再次提醒的是备份文件名不能有空格，否则DeferredJob报internal mysql error,please try again later or call us for help错误。如果报了call storage error的话，一般是storage名字问题。

状态查询

查询时先查询数据库中记录的状态，若全完成则不再发送邮件，否则再查询sae中执行情况，若全完成则发送邮件，否则返回当前任务状态。

定时任务

每天1点35执行备份，每天从8点-22点，每4个小时查询一次备份状态

目前不知道能否通过Cron实现条件定时任务，满足一定条件则不在继续定时任务之类的，或者其他方式实现也行，求指导！
